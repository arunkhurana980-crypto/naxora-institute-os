# Part 126 Honest Boundary

Part 126 creates real native MongoDB records for all nine Part 125 action types.

Historical module schemas may differ. Student references are discovered strictly inside the institute. Lead/Class references use an exact native record when safely found; otherwise a cleaned institute action reference is stored and the validation mode says so.

Email, SMS and WhatsApp are optional. Without a private provider, status is `provider_not_configured`, never “sent”.

Part 126 does not charge cards, transfer money or issue refunds.

Part 127 is still required for final role regression, production acceptance, deployment/rollback drill and launch decision.
