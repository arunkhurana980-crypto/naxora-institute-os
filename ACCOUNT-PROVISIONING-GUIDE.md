# Account Provisioning Guide

## Owner
Adopt a valid existing owner session first.

## Create users
Open:

```text
/account-access-manager
```

For each user enter:
- unique Login ID, email or phone inside the institute,
- display name,
- correct role,
- temporary password.

The server returns the required exact confirmation during preview.

## Share safely
Share the temporary password privately. Do not place it in:
- public WhatsApp groups,
- GitHub,
- screenshots,
- VANI/chat,
- institute notices.

The user must change it at first login.

## Disable
Disable an account when access should stop. Existing Part 120 sessions are revoked.

## Reset
A reset sets a temporary password, revokes previous Part 120 sessions and requires a new password at the next login.
